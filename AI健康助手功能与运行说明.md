# AI健康助手功能与运行说明

## 1. 当前已具备的功能点

> 以下同时包含“已上线能力”和“本轮文档新增的待开发能力说明”，用于指导下一阶段实现。

## 1.1 会话与问答

1. 支持发起多轮对话（患者自然语言提问）。
2. 支持会话列表展示与会话切换。
3. 支持会话历史消息读取（按 session_id 拉取）。
4. 问答结果会落入会话内存存储（重启服务后会丢失，当前为 V1 轻量方案）。

## 1.2 智能导诊

1. 支持“快速导诊”入口。
2. 导诊输出已按 `AI健康助手需求文档.md` 6.1 的模板严格约束，固定结构为：
   - 结论摘要
   - 风险等级（低/中/高）
   - 建议行动（最多3条）
   - 可选就诊科室
   - 免责声明
3. 命中高危词（如胸痛/呼吸困难等）时，优先给出急诊提示。

## 1.3 多模型路由（千问 + DeepSeek）

1. 高风险问题：优先路由到 Qwen。
2. 常规问题：优先路由到 DeepSeek。
3. 首选模型失败时自动回退到另一模型。
4. API key 通过环境变量/.env 读取，不写死在代码。

## 1.4 流式输出（SSE）

1. `AI Chat` 已支持流式输出（后端 `stream=1`）。
2. 前端逐字显示回答增量，减少“请求超时但后台已处理”的体验问题。
3. 流式结束时返回 done 事件，包含最终 answer/session_id/reply_id。

## 1.5 门诊动作联动

在 AI 页面提供快捷动作按钮：
1. 去挂号（跳转 `/outpatient/register`）
2. 看排队（跳转 `/outpatient/queue`）
3. 去支付（跳转 `/outpatient/payment`）
4. 刷新会话（手动拉取最新会话与消息）

## 1.6 新增功能（本轮规划）

### A. 个性化康复伴行

功能作用：
- 用户先选择指定病历，系统提取该病历字段，再结合用户单独输入提示词，生成：
  1) 用药提醒
  2) 复诊建议
  3) 注意事项说明

核心约束：
- 仅做康复管理建议，不做诊断结论，不改处方。
- 必须严格按照固定输出模板返回，便于前端稳定展示。
- `record_id` 必填；`user_prompt` 为单独输入字段（不与病历字段混写）。

### B. 检查报告“翻译官”

功能作用：
- 用户选择某个病历后，系统提取该病历的“诊断信息 + 检查治疗部分”，再结合用户单独输入提示词做非诊断型解释。
- 将专业术语翻译成大白话，提升患者理解度。

核心约束：
- 仅做信息翻译与解释，不做诊断、不替代医生面诊。
- 必须严格按固定模板输出。
- `record_id` 必填；`user_prompt` 为单独输入字段。

## 2. 后端接口清单

1. `POST /ai/chat`

2. `POST /ai/triage`
3. `GET /ai/sessions`

   - 处理：后端只提取该 `record_id` 对应病历字段拼装提示词

   - 处理：后端只提取该 `record_id` 对应病历字段拼装提示词
   - 输出：`translator_output`（严格模板）

## 5. 运行方式

## 5.1 配置模型密钥

推荐使用：`application/server/.env`

示例：

```env
AI_QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_QWEN_API_KEY=你的QwenKey
AI_QWEN_MODEL=qwen-plus

AI_DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
AI_DEEPSEEK_API_KEY=你的DeepSeekKey
AI_DEEPSEEK_MODEL=deepseek-chat
```

说明：
1. 后端已支持自动尝试加载：
   - 根目录 `.env`
   - `application/server/.env`
2. `.gitignore` 已忽略上述 .env，避免密钥误提交。

## 5.2 启动后端

在 `application/server` 目录执行：

```bash
go run main.go
```

默认会提供 `/api/v2/*` 接口。

## 5.3 启动前端

在 `application/web` 目录执行：

```bash
npm install
npm run dev
```

访问前端后，进入左侧菜单 `AI健康助手` 使用。

### D. 用户提问引导提示词（建议放前端输入框上方/placeholder）

#### 1) 导诊提问引导

```text
为了获得更准确建议，请尽量按这个格式描述：
1) 主要症状（例如：胸闷、咳嗽、发热）
2) 持续时间（多久了）
3) 严重程度（轻/中/重，是否加重）
4) 伴随症状（如气短、头晕、恶心）
5) 既往病史与用药（如高血压、糖尿病、正在服药）
6) 特殊情况（孕期、过敏史、近期手术等）
```

#### 2) 个性化康复伴行 提问引导

```text
为了生成更贴合你的康复建议，请尽量补充：
1) 当前最不舒服的症状
2) 症状持续多久、是否加重
3) 当前正在服用的药物与频次
4) 最近一次复诊/检查时间
5) 你最关心的康复问题（如“何时复诊最合适”）
```

#### 3) 检查报告“翻译官” 提问引导

```text
为了翻译得更准确，请告诉我：
1) 你想重点看哪部分（诊断/检查结果/治疗建议）
2) 哪些术语看不懂
3) 你最关心的一个问题（例如“严重吗”“要不要马上复查”）
4) 是否有医生当面给过补充说明
```

这些引导可显著提升模型回答质量，减少因信息缺失导致的泛化回答。

## 6. 常见问题排查

## 6.1 问题：请求超时

表现：页面提示 timeout，但刷新后看到回答。

原因：
- 模型响应慢于前端超时阈值。

已做优化：
1. AI API 独立加长 timeout。
2. 已接入流式输出，前端边收边渲染。

仍异常时检查：
1. 外网是否能访问模型 base_url。
2. API key 是否有效。
3. 服务端日志是否有 4xx/5xx。

## 6.2 问题：无模型回复

检查：
1. `.env` 是否生效（变量名是否正确）。
2. 模型名是否存在（如 qwen-plus / deepseek-chat）。
3. 是否触发了回退（主模型失败后应尝试备模型）。
